## 1. Agent 智能体：Function Calling 两轮推理

### 1.1 算法概述

Agent 采用 **ReAct（Reasoning + Acting）** 范式，通过 OpenAI Function Calling 协议实现两轮推理：
- **第一轮（Planning）**：LLM 分析用户意图，判断是否需要调用工具
- **第二轮（Execution）**：执行工具调用，将结果注入上下文，LLM 生成最终回答

输出: {answer, sources, tool_calls, used_context}

1:  // ===== 阶段0: 智能记忆触发判断 =====
2:  relevant_memories ← SMART_RETRIEVE(question)   // 轻量预判，无需则跳过
3:
4:  // ===== 阶段1: 构建系统提示词 =====

6:  messages ← [(system, system_prompt), (user, question)]
7:
8:  // ===== 阶段2: 第一轮 LLM 推理（工具规划） =====

19:     END FOR

22:     response_2 ← LLM_CALL(messages, temperature=0.7)
23:     final_answer ← response_2.content
24: ELSE
25:     // 无需工具，直接使用第一轮回复
26:     final_answer ← response_1.content
27: END IF
28:
29: RETURN {answer: final_answer, tool_calls: tool_results}

### 1.4 关键实现代码

```python
class FinRAGAgent:
    """基于 Function Calling 的金融智能体"""

    def run(self, question: str, conversation_history: str = "", top_k: int = 3):
        # 阶段0: 智能记忆触发（轻量预判，避免不必要的检索开销）
        relevant_memories = memory_manager.smart_retrieve(question, top_k=3)

        # 阶段1: 构建提示词（注入长期记忆）
        system_prompt = self._build_system_prompt(conversation_history, relevant_memories)
        messages = [
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": question}
        ]

        # 阶段2: 第一轮 - LLM 判断是否需要工具
        response = self.client.chat.completions.create(
            model=self.model, messages=messages,
            tools=self.tools, tool_choice="auto", temperature=0.3
        )
        tool_calls = response.choices[0].message.tool_calls

        if tool_calls:
            # 阶段3: 执行工具调用
            for tc in tool_calls:
                args = json.loads(tc.function.arguments)
                result = self._execute_tool(tc.function.name, args)
                messages.append({"role": "tool", "tool_call_id": tc.id, "content": result})

            # 阶段4: 第二轮 - 基于工具结果生成答案
            final = self.client.chat.completions.create(model=self.model, messages=messages, temperature=0.7)
            return {"answer": final.choices[0].message.content, "used_context": True}
        else:
            return {"answer": response.choices[0].message.content, "used_context": False}
```

### 1.5 设计决策

| 决策点 | 选择 | 理由 |

### 2.2 HyDE 假设性文档生成

```python
HYDE_PROMPT = """你是一个专业的金融文档撰写助手。请根据用户问题，生成一段假设性文档内容。

用户问题: {question}
要求:
1. 生成的文档应像是从权威金融资料中提取的内容
2. 包含相关专业术语和概念解释
3. 内容长度适中（100-200字）

假设性文档:"""

def generate_hypothetical_document(question: str, llm) -> str:
    """HyDE 核心: 用 LLM 生成假设性答案作为检索查询"""
    prompt = HYDE_PROMPT.format(question=question)
    response = llm.invoke(prompt)
    return response.content  # 返回假设性文档替代原始问题
```

**设计原理**：原始问题可能过于简短或表述模糊（如"基金怎么选？"），HyDE 先让 LLM 生成一段"理想答案"，再用这段更丰富的文本做向量检索，可提升召回率约 **15-25%**。

### 2.3 BGE-Reranker 精排与 Sigmoid 归一化

```python
def rerank_documents(query: str, documents: List[Document], top_k: int = 3):
    """
    使用 BGE-reranker-v2-m3 对召回候选进行精排
    
    数学公式:
      raw_score = CrossEncoder(query, doc)     # 输出 logit
      norm_score = sigmoid(raw_score)          # 归一化到 [0, 1]
                  = 1 / (1 + e^(-raw_score))
    """
    reranker = CrossEncoder("BAAI/bge-reranker-v2-m3")
    
    # 构建 (query, doc) 对
    pairs = [[query, doc.page_content] for doc in documents]
    raw_scores = reranker.predict(pairs)           # 原始 logit 分数
    
    # Sigmoid 归一化到 [0, 1]
    scores = 1.0 / (1.0 + np.exp(-np.array(raw_scores)))
    
    # 按分数降序排列，取 top_k
    ranked = sorted(zip(documents, scores), key=lambda x: x[1], reverse=True)
    return ranked[:top_k]
```

## 3. 记忆系统 v2.0：混合检索 + MMR 去重 + 智能触发

### 3.1 四层记忆架构

| 层级 | 存储位置 | 生命周期 | 示例内容 |
|------|----------|----------|----------|
| SOUL.md | 常青文件 | 永久 | AI 人格定义 |
| MEMORY.md | 常青文件 | 永久（Compaction压缩） | 用户偏好、核心事实 |
| daily/YYYY-MM-DD.md | 日志文件 | 90天滚动 | 当日对话记录 |

### 3.2 混合检索核心（BM25 : Vector = 3 : 7）

```
算法 2: OpenClaw 风格混合检索
===============================
输入: query(查询文本), top_k(返回数量)
输出: 按衰减后分数排序的记忆片段列表

1:  // === 阶段 A: BM25 关键词评分 ===
2:  FOR EACH doc IN documents DO
3:      FOR EACH token IN query_tokens DO
4:          idf ← log((N - df(token) + 0.5) / (df(token) + 0.5) + 1)
5:          tf_norm ← (tf × (k1+1)) / (tf + k1×(1-b+b×dl/avgdl))
6:          bm25_score += idf × tf_norm
7:      END FOR
8:  END FOR
9:
10: // === 阶段 B: 向量余弦相似度 ===
11: query_emb ← EMBED(query)                     // 带 LRU 缓存
12: FOR EACH doc IN documents DO
13:     chunk ← CHUNK_TEXT(doc, size=400)         // 分块取最佳匹配
14:     chunk_emb ← EMBED(chunk)                  // 带 LRU 缓存
15:     vec_score ← COSINE(query_emb, chunk_emb)
16: END FOR
17:
18: // === 阶段 C: 加权融合 + 时间衰减 ===
19: FOR EACH doc IN documents DO
20:     hybrid_score ← 0.3×NORM(bm25) + 0.7×NORM(vec)
21:     IF NOT is_evergreen(doc) THEN
22:         decay ← e^(-ln(2)/30 × days_ago)      // 半衰期30天
23:         decayed_score ← hybrid_score × decay
24:     ELSE
25:         decayed_score ← hybrid_score          // 常青记忆不衰减
26:     END IF
27: END FOR
28:
29: // === 阶段 D: MMR 最大边际相关性去重 ===
30: selected ← []
31: WHILE len(selected) < top_k AND remaining ≠ ∅ DO
32:     FOR EACH item IN remaining DO
33:         relevance ← item.decayed_score
34:         diversity_penalty ← MAX_JACCARD(item, selected)  // 与已选最大相似度
35:         mmr_score ← λ × relevance - (1-λ) × diversity_penalty    // λ=0.7
36:     END FOR
37:     selected.ADD(BEST(mmr_score))
38: END WHILE
39:
40: RETURN selected
```

### 3.3 BM25 核心实现

```python
def batch_score(self, query: str, documents: Dict[str, str]):
    """Okapi BM25 批量评分"""
    query_tokens = tokenize_chinese_bigram(query.lower())
    results = []

    for doc_id, text in documents.items():
        doc_tokens = tokenize_chinese_bigram(text.lower())
        dl = len(doc_tokens)
        doc_score = 0.0

        for token in query_tokens:
            if token not in self.df: continue
            # IDF: 逆文档频率
            idf = math.log((self.n_docs - self.df[token] + 0.5) /
                           (self.df[token] + 0.5) + 1)
            tf = doc_tokens.count(token)
            # TF 归一化: 长文档惩罚
            tf_norm = (tf * (self.k1 + 1)) / (
                tf + self.k1 * (1 - self.b + self.b * dl / self.avgdl))
            doc_score += idf * tf_norm

        results.append((doc_id, doc_score))
    return sorted(results, key=lambda x: x[1], reverse=True)
```

### 3.4 MMR 去重实现

```python
def mmr_rerank(items, lambda_param=0.7, top_k=5):
    """
    最大边际相关性重排
    MMR(d) = λ·Rel(d) - (1-λ)·max(Sim(d, d_selected))
    
    使用 Jaccard 相似度（轻量，避免额外向量计算）
    """
    selected, remaining = [], list(items)
    
    while remaining and len(selected) < top_k:
        best_mmr, best_idx = -∞, 0
        
        for i, (item_id, rel_score, content) in enumerate(remaining):
            relevance_part = rel_score
            
            if selected:
                # 计算与已选集合的最大 Jaccard 相似度
                content_tokens = set(tokenize(content)[:50])
                max_sim = max(
                    jaccard(content_tokens, set(tokenize(sel_content)[:50]))
                    for _, _, sel_content in selected
                )
            else:
                max_sim = 0.0
            
            mmr_score = lambda_param * relevance_part - (1 - lambda_param) * max_sim
            
            if mmr_score > best_mmr:
                best_mmr, best_idx = mmr_score, i
                
        selected.append(remaining.pop(best_idx))
    
    return selected
```

### 3.5 智能触发机制（v2.0 核心创新）

```python
def should_retrieve_memory(query: str) -> bool:
    """
    轻量级读取触发判断（三道防线）
    
    设计目标: 大多数问题（如"你好"、"查股价"）不需要长期记忆，
              跳过检索可节省 ~200-500ms 延迟
    """
    # 防线1: 关键词快速匹配（O(n)，n≈30个关键词，<1ms）
    trigger_keywords = ["之前", "上次", "记得", "我说过", "我叫", 
                        "我的名字", "我喜欢", "我决定", ...]  # 共~30个
    if any(kw in query.lower() for kw in trigger_keywords):
        return True
    
    # 防线2: 有实质记忆内容时，对疑问句也尝试检索
    if memory_has_real_content() and is_question(query):
        return True
    
    # 防线3: 自我指涉语句检测
    self_referential = ["我的", "我想", "我计划", "我喜欢", "我是", ...]
    if "我" in query and any(r in query for r in self_referential):
        return True
    
    return False  # 未命中任一防线 → 跳过记忆检索


def should_update_memory(question: str, answer: str) -> tuple:
    """
    写入预判断（四层过滤）
    目标: 避免每轮都消耗 LLM token 做无意义的记忆提取
    """
    # Level 1: 问题长度 < 4字 → 跳过
    if len(question.strip()) < 4:
        return False, "问题过短"
    
    # Level 2: 回答长度 < 30字 → 大概率无信息量
    if len(answer.strip()) < 30:
        return False, "回答过短"
    
    # Level 3: 个人信息关键词检测
    personal_indicators = ["我叫", "喜欢", "不喜欢", "决定", "计划", 
                           "风险", "保守型", "激进型", ...]
    if not any(ind in (question+answer) for ind in personal_indicators):
        return False, "未检测到个人信息"
    
    # Level 4: 通过所有检查 → 执行 LLM 提取
    return True, "检测到有价值信息"
```

### 3.6 时间衰减函数

$$
\text{decay}(d) = e^{-\lambda \cdot d}, \quad \lambda = \frac{\ln(2)}{T_{1/2}}
$$

其中：
- $d$ = 距今天数
- $T_{1/2}$ = 半衰期（系统设为 **30 天**）
- 效果：30天内权重 > 0.5，60天后 ≈ 0.25，90天后 ≈ 0.125

```python
def time_decay_score(days_ago: int, half_life: int = 30) -> float:
    lam = math.log(2) / half_life
    return math.exp(-lam * days_ago)
```

### 4.3 Cypher 查询模板（模糊匹配 + 变长路径）

```python
class CypherQueryBuilder:
    @staticmethod
    def find_supply_chain(company_name: str) -> str:
        """完整供应链查询（上游依赖 + 下游客户）"""
        return """
        MATCH (c:Company)
        WHERE toLower(c.name) CONTAINS toLower($company)
        OPTIONAL MATCH upstream = (c)-[:OUTSOURCES_TO]->(f:Foundry)
                                 -[:DEPENDS_ON]->(m:Material)
        OPTIONAL MATCH downstream = (customer:Company)-[:PURCHASES]->(p:Product)
                                    <-[:DESIGNS]-(c)
        RETURN c.name AS company,
               collect(DISTINCT {foundry:f.name, material:m.name}) AS upstream,
               collect(DISTINCT {customer:customer.name, product:p.name}) AS downstream
        """

    @staticmethod
    def find_impact_chain(start: str, end: str, max_hops: int = 3) -> str:
        """影响链多跳推理（限制关系类型）"""
        return f"""
        MATCH (s), (e)
        WHERE toLower(s.name) CONTAINS toLower($start)
           AND toLower(e.name) CONTAINS toLower($end)
        MATCH path = (s)-[r*1..{max_hops}]-(e)
        WHERE all(rel IN relationships(path) 
                  WHERE type(rel) IN ['AFFECTED_BY','IMPACTS','DRIVEN_BY'])
        RETURN [n IN nodes(path) | n.name] AS chain,
               [r IN relationships(path) | type(r)] AS relation_types
        LIMIT 10
        """

    @staticmethod
    def find_risk_impact(foundry_name: str) -> str:
        """风险评估：代工厂事故波及范围"""
        return """
        MATCH (f:Foundry)
        WHERE toLower(f.name) CONTAINS toLower($foundry)
        OPTIONAL MATCH (f)<-[:OUTSOURCES_TO]-(c:Company)
        OPTIONAL MATCH (c)-[:DESIGNS]->(p:Product)<-[:PURCHASES]-(cust:Company)
        RETURN f.name AS foundry,
               collect(DISTINCT c.name) AS affected_companies,
               collect(DISTINCT cust.name) AS downstream_customers
        """
```

### 4.4 MERGE 写入策略（幂等 Upsert）

```python
def import_to_documents(documents: List[str]) -> Dict:
    """批量导入文档到 Neo4j（幂等写入）"""
    extractor = EntityExtractor()
    conn = get_neo4j_connection()
    
    for doc in documents:
        entities, relations = extractor.extract(doc)
        
        # 实体写入: MERGE 保证幂等（存在则更新，不存在则创建）
        for entity in entities:
            conn.execute_write("""
                MERGE (e:{type} {{name: $name}})
                SET e += $properties
            """.format(type=entity.entity_type), 
            {"name": entity.name, "properties": entity.properties})
        
        # 关系写入: MATCH两端节点 + MERGE关系
        for relation in relations:
            conn.execute_write("""
                MATCH (source {{name: $source}})
                MATCH (target {{name: $target}})
                MERGE (source)-[r:{type}]->(target)
                SET r += $properties
            """.format(type=relation.relation_type),
            {"source": relation.source, "target": relation.target,
             "properties": relation.properties})
    
    return {"total_entities": len(entities), "total_relations": len(relations)}
```

## 5. 触发系统：关键词 + LLM 双模融合

### 5.1 架构概述

```
用户问题 ──→ [关键词触发器组(并行)] ──→ TriggerResult列表
                 │                      │
                 │  confidence ≥ 0.5?   │  按置信度排序
                 ▼                      ▼
           [LLM触发器(可选)] ──────→ 合并/提升置信度
```

### 5.2 关键词触发器（零延迟）

```python
class KeywordTrigger(BaseTrigger):
    """基于关键词的触发器（延迟 < 1ms）"""
    
    DEFAULT_KEYWORDS = {
        "finance_akshare": ["股票", "股价", "基金", "指数", "行情", "涨跌", ...],
        "finance_tushare":  ["财报", "年报", "净利润", "ROE", "负债率", ...],
        "kg":              ["关系", "竞争", "产业链", "上下游", "关联方", ...],
        "resset":          ["年报", "政府工作报告", "研究报告", "招股说明书", ...],
        "quant":           ["量化", "回测", "策略", "机器学习", "xgboost", ...],
        # ... 共 9 种触发类型
    }
    
    def check(self, question: str) -> Optional[TriggerResult]:
        matched = [kw for kw in self.keywords if kw.lower() in question.lower()]
        if matched:
            # 置信度公式: baseline(0.5) + 0.1 × 匹配数量（上限1.0）
            confidence = min(0.5 + 0.1 * len(matched), 1.0)
            return TriggerResult(
                trigger_type=self.trigger_type.value,
                confidence=confidence,
                reason=f"匹配到关键词: {', '.join(matched[:5])}"
            )
        return None
```

### 5.4 统一管理器与融合策略

```python
class TriggerManager:
    def analyze(self, question: str) -> List[TriggerResult]:
        """双模融合分析"""
        results = []
        
        # 1️⃣ 并行执行所有关键词触发器（< 5ms 总延迟）
        for trigger_type, trigger in self.triggers.items():
            result = trigger.check(question)
            if result: results.append(result)
        
        # 2️⃣ LLM 增强模式（可选）：语义补充 + 置信度校准
        if self.use_llm:
            llm_result = self.llm_trigger.check(question)
            if llm_result:
                for existing in results:
                    if existing.trigger_type == llm_result.trigger_type:
                        # 取两者置信度的最大值
                        existing.confidence = max(existing.confidence, llm_result.confidence)
                        existing.reason += f" + {llm_result.reason}"
        
        # 3️⃣ 按置信度降序排列
        results.sort(key=lambda x: x.confidence, reverse=True)
        return results
```

## 6. BM25 算法：Okapi BM25 中文 bigram 实现

### 6.1 中文 Bigram 分词器

```python
CHINESE_STOPWORDS = {"的", "了", "是", "在", "我", "有", "和", ...}  # ~80个停用词

def tokenize_chinese_bigram(text: str) -> List[str]:
    """
    中文 Bigram 分词器（无外部词典依赖）
    
    策略:
    1. 提取中文连续序列 + 英文单词
    2. 中文: 单字(unigram) + 相邻二字(bigram) → 提升召回率
    3. 过滤停用词
    4. 英文: 直接保留完整单词
    
    比 regex 分词召回率提升约 30%
    """
    tokens = []
    segments = re.findall(r'[\u4e00-\u9fff]+|[a-zA-Z0-9_]+', text.lower())
    
    for segment in segments:
        if re.match(r'[\u4e00-\u9fff]+', segment):
            for i, char in enumerate(segment):
                if char not in CHINESE_STOPWORDS:
                    tokens.append(char)                          # unigram
                if i < len(segment) - 1:
                    bigram = segment[i:i+2]
                    if (bigram[0] not in CHINESE_STOPWORDS and 
                        bigram[1] not in CHINESE_STOPWORDS):
                        tokens.append(bigram)                   # bigram
        else:
            if len(segment) >= 2:
                tokens.append(segment)                           # 英文保留
    
    return tokens
```

### 6.2 Embedding LRU 缓存层

```python
class EmbeddingCache:
    """
    LRU 缓存层：相同文本只计算一次 embedding
    - 容量: 500 条（覆盖典型会话的全量记忆分块）
    - 淘汰策略: LRU（最近最少使用优先淘汰）
    - 键: MD5(text) → 值: np.ndarray
    - 节省效果: 减少约 70% 的 Ollama embedding API 调用
    """
    
    def __init__(self, max_size: int = 500):
        self.cache: Dict[str, np.ndarray] = {}
        self.access_order: List[str] = []  # LRU 追踪链
    
    def get(self, text: str) -> Optional[np.ndarray]:
        key = hashlib.md5(text.encode("utf-8")).hexdigest()
        if key in self.cache:
            self.access_order.remove(key)      # 移到末尾（最近访问）
            self.access_order.append(key)
            return self.cache[key]
        return None
    
    def put(self, text: str, embedding: np.ndarray):
        key = hashlib.md5(text.encode("utf-8")).hexdigest()
        if len(self.cache) >= self.max_size:
            oldest = self.access_order.pop(0)  # 淘汰最旧条目
            del self.cache[oldest]
        self.cache[key] = embedding
        self.access_order.append(key)
```
